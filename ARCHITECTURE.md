# NovoEd Component Toolkit — Architecture & Development Guide

## System Overview

The toolkit has 3 layers:
1. **JSX Artifact** (`NovoEd_Component_Library.jsx`) — React component that renders in Claude's chat. Has AI generation built-in.
2. **Standalone HTML** (`index.html`) — Vanilla JS version for GitHub Pages. Zero dependencies. Built from the JSX core logic.
3. **Build Script** (`build_vanilla.py`) — Python script that extracts the core JS from the JSX and wraps it in a vanilla HTML app.

## File Structure

```
NovoEd_Component_Library.jsx    ← The source of truth (React)
build_vanilla.py                ← Converts JSX core → standalone HTML
index.html                      ← Output: standalone HTML for GitHub Pages
README.md                       ← GitHub repo README
NovoEd_Toolkit_Claude_Project_Instructions.md  ← Claude Project system prompt
```

## Architecture

### JSX File Structure (NovoEd_Component_Library.jsx)

```
Lines 1-13:     Brand tokens (B), Froala class (FR), escape function (esc)
Lines 14-46:    Component registries (HTML_COMPS, SCORM_COMPS)
Lines 47-183:   genHTML() — generates static HTML for 16 HTML components
Lines 184-213:  DEFAULTS — default data for all 26 components
Lines 214-218:  scTag() helper — builds <script> tags safely
Lines 219-502:  genSCORMhtml() — generates interactive HTML for 10 SCORM components
Lines 503-552:  createZip() — inline ZIP creator (no JSZip dependency)
Lines 553-571:  downloadSCORM() — generates SCORM zip with imsmanifest.xml
Lines 572-587:  BrandSwitch React component
Lines 588+:     Main App React component (state, screens, editor, preview)
```

### Core Logic (Lines 1-571) — PURE JS, no React
This is extracted by build_vanilla.py and shared between JSX and HTML versions.

Contains:
- Brand tokens
- Component registries  
- HTML generator (genHTML)
- SCORM generator (genSCORMhtml) 
- Default data
- ZIP creator
- SCORM downloader

### React UI (Lines 572+) — JSX ONLY
- BrandSwitch component
- App component with useState hooks
- Home screen (banner, search, AI generator, tool cards)
- List screen (search, component grid)
- Editor screen (form fields, preview, code output)

### Vanilla UI (inside build_vanilla.py)
- Plain JS equivalent of the React UI
- Uses innerHTML rendering instead of React
- State management via plain object + render() function
- CSS hover classes instead of onMouseEnter/onMouseLeave

## Key Concepts

### Brand System
```js
const B = {
  bcg:  { pri:"#29BA74", priDk:"#1B7A4F", priLt:"#E6F7EF", ... },
  bcgu: { pri:"#197A56", priDk:"#0D3B2C", priLt:"#E8F3ED", ... },
};
```
Every component uses `b.pri`, `b.priDk`, etc. Brand switching just changes which key is active.

### HTML Components
- Generated by `genHTML(id, brand, data)`
- Return raw HTML strings using `<table>` tags (for email/Froala compatibility)
- Use inline styles only (no CSS classes) because NovoEd's Froala editor strips classes
- Use `class="froala-style-subtitle"` for headings (NovoEd recognizes this)

### SCORM Components  
- Generated by `genSCORMhtml(sel, data, brand)`
- Return full HTML documents (<!DOCTYPE html>...)
- Have CSS animations, transitions, backdrop-filter
- Include inline JavaScript for interactivity
- Use `scTag()` helper to avoid `<script>` tag issues in HTML embedding
- Background controlled by `data.bg` field ("gradient" or "none")

### SCORM Download
- `createZip()` — pure JS ZIP file creator, no external libraries
- `downloadSCORM()` creates a zip with:
  - `index.html` — the interactive HTML
  - `imsmanifest.xml` — SCORM 1.2 manifest

### Script Tag Safety
SCORM HTML contains `<script>` tags for interactivity. When this JS is embedded inside the standalone HTML's own `<script>` block, the browser's HTML parser would break on `</script>`. 

Solution: `scTag(code)` function builds script tags via string concatenation:
```js
function scTag(code) { return '<scr'+'ipt>'+code+'</scr'+'ipt>'; }
```
The build script also escapes `<\/script>` → `<\/scri"+"pt>` in the core JS.

### Conditional Fields
- If `data.title` exists and is empty string → nothing renders (user cleared it)
- If `data.title` doesn't exist → falls back to component name
- Same for `data.body`
- Editor shows fields based on what properties exist in the data object

### Image Support
- Flip cards and stacked cards have `img` field per item
- If `img` URL is provided → renders `<img>` tag
- If empty → falls back to numbered badge or icon
- Images can be NovoEd embedding URLs or any hosted image

## How to Make Changes

### Adding a New HTML Component

1. Add to `HTML_COMPS` array:
```js
{ id:"mycomp", n:"My Component", d:"Description", ic:"★" },
```

2. Add default data to `DEFAULTS`:
```js
mycomp: { items:[{title:"Item 1",desc:"Description"}] },
```

3. Add generator in `genHTML()`:
```js
if (id === "mycomp") {
  // Build HTML string using b.pri, b.tx, etc.
  return '<table>...</table>';
}
```

### Adding a New SCORM Component

1. Add to `SCORM_COMPS` array:
```js
{ id:"s_mycomp", n:"My Interactive", d:"Description", ic:"★" },
```

2. Add default data to `DEFAULTS`:
```js
s_mycomp: { title:"Title", body:"Subtitle", items:[{title:"Item 1",desc:"Desc"}] },
```

3. Add generator in `genSCORMhtml()`:
```js
} else if (sel === "s_mycomp") {
  h += '/* CSS styles */';
  h += '</style></head><body><div class="container">';
  h += pageHeader;  // Uses title/body if not empty
  // Build interactive HTML
  // For JavaScript, use scTag():
  h += scTag('function myFunc(){...}');
  h += '</body></html>';
}
```

### Rebuilding the Standalone HTML

After modifying the JSX:

```bash
# 1. Extract core logic (everything before "/* ── Brand Switcher */")
# Find the line number:
grep -n "^/\* ── Brand Switcher" NovoEd_Component_Library.jsx
# Extract (replace 578 with actual line number - 1):
sed -n '2,578p' NovoEd_Component_Library.jsx > core.js

# 2. Run build script
python3 build_vanilla.py

# 3. Copy to index.html for GitHub
cp NovoEd_Component_Library.html index.html
```

### Important: Vanilla Build Sync

The vanilla build (build_vanilla.py) has its OWN UI code for:
- Home page layout
- List screen  
- Editor screen
- Button hover classes

If you change the JSX UI (screens, layout, new fields), you MUST also update the corresponding section in build_vanilla.py. The core logic (components, generators, defaults) is shared automatically.
